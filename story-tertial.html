The Problem

Every tertial review required manually opening the Jira Portfolio Plan, filtering by T2/T3, and copying epic statuses into slides. Stakeholders had no self-serve view of progress against tertial goals.

🔴 The preparation loop 30–60 minutes before each stakeholder meeting: open Portfolio Plan, filter T2/T3 epics, check story counts, copy Customer Value and Business Impact from each epic, assemble a progress slide. Every time.

The Data Challenges

Getting the right 76 epics in the right order took three iterations. Each failure taught something important about how this Jira instance works.

Attempt 1 — Wrong

Filter by creation date

Result: Only 4 T2 / 46 T3 epics. Creation date doesn’t correlate with which tertial an epic belongs to — many old epics carry forward.

Attempt 2 — Better

Parse title text (FY26-T2 / FY26-T3)

Result: 23 T2 / 23 T3 epics. More accurate but missed epics named differently and counted some that were T1 or T4.

Attempt 3 — Correct

Use the Planned Tertial custom field

Jira field customfield_13202 stores the official Planned Tertial value. Result: 20 T2 / 22 T3 via field + Kiosk-label fallback = 76 total.

🟢 Also fixed: story fetching Stories must be fetched with "Epic Link" = KEY JQL — not parent = KEY. The parent field returns 0 results on this Jira instance.

Building It

Step 1

Fetch all T2+T3 epics

JQL: "Planned Tertial" in ("FY26 - T2","FY26 - T3") + Kiosk label fallback for epics without the field set. 76 epics total.

Step 2

Enrich with CV, BI, and stories

Fetched customfield_24001 (Customer Value) and customfield_24000 (Business Impact) for each epic. Fetched stories via "Epic Link" = KEY.

Step 3

Apply Portfolio priority rank

Used ORDER BY Rank ASC in JQL to get the same priority order as the Jira Portfolio Plan view. Each epic shows its #N rank badge.

Step 4

T3 first, always-visible CV+BI

T3 (current tertial) displayed above T2. Customer Value and Business Impact shown outside any collapsible — always visible without clicking. Stories behind an expandable toggle.

How Priority Works

Epics are sorted by the same rank used in the official Jira Portfolio Plan:

🔵 Portfolio Plan URL https://jira.digital.ingka.com/secure/PortfolioPlanView.jspa?id=1000&sid=1042&vid=4191

The dashboard’s #1, #2, #3 … priority badges match exactly what the PO sees in the Portfolio Plan backlog view.
“The numbers on the dashboard match exactly what I see in the Portfolio Plan. Finally one source of truth.” — Vishal, after adding portfolio rank sorting
