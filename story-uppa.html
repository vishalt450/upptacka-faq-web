The Problem

Customers and co-workers constantly ask the same questions about the Upptäcka kiosk: “Can it order delivery?”, “Does it show PTAG prices?”, “What happens when I scan an item?”

🔴 The manual loop Answers were spread across 111 product handbook slides, a Confluence FAQ, and hundreds of Jira tickets. Finding one answer meant opening 3–4 different tools, each time.

Knowledge Sources

Uppa combines three sources, each ranked and scored per query:

📚

Curated FAQ

Hand-crafted Q&A pairs covering the most common kiosk questions. Highest confidence answers.

📄

111 Handbook Slides

Upptäcka product handbook fully parsed and indexed. Covers features, flows, and edge cases.

🆕

Jira Done Tickets

188 completed Jira tickets from INTERACT. Parsed for “What it does / Why it matters” sections.

🌐

Zero Backend

All knowledge is embedded in a single knowledge.json file. No server, no API calls, no cost per query.

3

Building It

Step 1

Start with FAQ only

Basic keyword matching against curated Q&A pairs. Worked for exact questions, failed on paraphrasing.

Step 2

Add NLP scoring

Added stop-word removal, stemming (Porter), and synonym expansion. Now “Can I place an order?” matches “delivery services ordering”.

Step 3

Ingest 111 handbook slides

Parsed the full Upptäcka handbook PDF. Each slide became a scored knowledge entry with section, title, and body text.

Step 4

Add Jira Done tickets via API

Fetched 188+ Done tickets from INTERACT via Jira REST API. Parsed Jira markup to extract clean text. Removed boilerplate (Definition of Done, empty sections).

Step 5

Google-style autocomplete

Added live autocomplete suggestions as the user types, using the combined title index from all sources.

4

How Search Works

Every query goes through a multi-stage scoring pipeline:

🔵 Scoring pipeline 1. Tokenise query → remove stop words → stem tokens
2. Expand with synonyms (e.g. “order” → “delivery, purchase, buy”)
3. Score each knowledge entry: exact phrase match (3×) + word overlap + title bonus
4. Return top-3 results with source card + confidence badge
“It found the answer in the handbook even when I typed it completely differently from how it was written.” — Vishal, testing with real customer questions
