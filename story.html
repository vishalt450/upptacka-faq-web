The Problem

As the Product Owner for Upptäcka — IKEA’s in-store kiosk — answering questions took too long. Customers and co-workers constantly asked: “What can the kiosk do?” — and the answer lived scattered across 111 handbook slides, 282 Jira tickets, and FAQ documents.

🔴 The manual loop Every question meant digging through Confluence, Jira, or asking a developer. Stakeholder meetings had no single view of epic progress. Usage data lived in a Looker Studio dashboard only the data team could update.

Three Tools Built

Each tool solves a specific problem for the Upptäcka team and stakeholders.

🤖

Uppa — AI Chatbot

Customers and co-workers ask questions about the kiosk in natural language. Uppa searches FAQ, 111 handbook slides, and 188+ Jira Done tickets to give a clean answer with source cards.

📈

KPI Dashboard

Live data from BigQuery (ingka-upptacka-data-prod). Sessions, value added, conversion funnel, PTAG, Sälja handovers, top 20 markets — refreshed on demand, ~$0.00001 per query.

📋

Tertial Progress Dashboard

76 epics from Jira INTERACT classified by Planned Tertial field, sorted by Portfolio priority rank. Customer Value + Business Impact always visible. T3 first.

📄

Confluence Pages

All three tools documented here in the Product AI Workshop space, with IKEA-network-protected links and refresh instructions for any team member.

3

The Journey

Everything was built iteratively in one session using GitHub Copilot CLI — no IDE, no build pipeline, just a terminal.

Step 1

Uppa chatbot — FAQ + handbook search

Started with a simple FAQ page. Added 111 handbook slides as a second knowledge source. Built NLP scoring (stem, synonyms, stop words) to find the best match.

Step 2

Jira as a knowledge source

Fetched 282 Done tickets from Jira INTERACT. Parsed Jira template markup to extract clean “What it does / Why it matters” answers. Built Google-style autocomplete.

Step 3

Tertial dashboard — Jira epics

Fetched 76 epics using the Planned Tertial custom field. Fixed Epic Link story fetching. Sorted T3 by Portfolio Plan rank. Customer Value + Business Impact always visible.

Step 4

KPI dashboard — BigQuery live data

Installed gcloud, authenticated with IKEA Google account, connected to ingka-upptacka-data-prod. Built conversion funnel, sparklines, PTAG, country table. Cost: $0.00001 per refresh.

Step 5

Published to IKEA network

Created repo on git.build.ingka.ikea.com. Enabled GitHub Pages — SSO protected, IKEA employees only. Created Confluence pages under Product AI Workshop.

4

The Data Sources

📊

BigQuery — ingka-upptacka-data-prod

Table: data_product.dp_usage_by_store_aggregation. Daily partitioned, 85 MB, 643 stores globally. Nightly exports from all kiosks.

🆕

Jira INTERACT

282 Done tickets (knowledge), 76 T2+T3 epics (progress), sorted by Planned Tertial field + Portfolio rank. Epic Link used for story fetching.

📚

111 Handbook Slides

Upptäcka product handbook embedded in the chatbot as a knowledge source. Scored and ranked per query.

🤖

FAQ Knowledge Base

Curated Q&A pairs covering the most common kiosk questions, with NLP synonym matching and stop-word filtering.

🟢 Cost safety Each full KPI dashboard refresh scans only 2.2 MB from the 85 MB partitioned table. Cost is ~$0.00001 per refresh — $0.001 for 100 refreshes. The table’s DATE partition means only the relevant date range is read.
