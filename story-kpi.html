The Problem

Upptäcka usage data existed in Looker Studio — but only the data team could refresh it, and it required navigating through a GCP project most POs had never opened.

🔴 The access gap Sessions, value-add, conversion funnel, PTAG performance — all available in BigQuery, but inaccessible to the PO without data team involvement. Monthly reviews were the only time numbers got shared.

The Data Stack

Upptäcka’s kiosk events flow through a well-defined data pipeline:

🔵 Pipeline Kiosk events (KEPS) → nightly batch export → BigQuery (ingka-upptacka-data-prod) → Looker Studio

The dashboard taps directly into the BigQuery layer, bypassing Looker Studio entirely.
📊

Project: ingka-upptacka-data-prod

The Upptäcka data product in GCP. Managed by the Upptäcka data team.

📋

Table: dp_usage_by_store_aggregation

Daily aggregated metrics per store: sessions, orders, value, funnel steps, PTAG events. DATE-partitioned.

📅

85 MB total, 2.2 MB per query

Partition pruning means only the relevant date range is scanned — keeping costs near zero.

🌐

643 stores globally

Every IKEA store with an active Upptäcka kiosk contributes daily data to the table.

Building It

Step 1

Discover the data product

Found ingka-upptacka-data-prod via the BigQuery Agents Hub. Confirmed table schema and partition structure. Cost-validated with a dry-run query: $0.00001 per full refresh.

Step 2

Authenticate with gcloud

Installed Google Cloud SDK via Homebrew. Used Application Default Credentials (gcloud auth application-default login) with the IKEA Google account.

Step 3

Build the Python data script

Wrote build_kpi.py: queries last 30d vs prev 30d, 16-week trend, PTAG funnel, conversion funnel, top-20 markets. Outputs kpi_data.json.

Step 4

Generate static HTML dashboard

Embedded the JSON data into a single-file HTML page with inline SVG sparklines, KPI cards with trend arrows, conversion funnel bars, and country table.

Step 5

Publish to IKEA GitHub Pages

Pushed to git.build.ingka.ikea.com/viton2/upptacka-dashboards. SSO-protected via SAML Auth0 — only IKEA employees can view it.

4

Cost & Safety

🟢 Verified cost: $0.00001 per full refresh BigQuery charges $5/TB scanned. DATE partitioning means each 30d query reads only 2.2 MB from the 85 MB table. 1,000 refreshes costs $0.01.
🔵 Access control The dashboard is hosted on IKEA internal GitHub Pages (SAML/SSO protected). No data leaves IKEA systems. BigQuery credentials use Application Default Credentials — no hardcoded keys anywhere.
“I ran a dry-run first. Zero cost confirmed before we touched live data.” — Vishal, on the BigQuery cost check
